如何把三個網站「優雅整合」？

保留既有 GitHub Pages 網址，只做一個 總入口（Hub）。視覺採極簡設計、清楚分塊、明顯行動按鈕。下面提供落地做法。

設計原則

  • ✓清楚層級：Hero ▶ 卡片網格 ▶ 功能/FAQ ▶ 頁尾
  • ✓一致的按鈕與留白，行動導向
  • ✓外部內容以新視窗開啟，避免跳出主頁
  • ✓用 CSS 變數 管理色彩，快速換色

部署步驟（GitHub Pages）

建立新倉庫

Repo 名稱例如 anemia-hub，公開或私有皆可。

新增本檔 index.html

把這個檔案存成 index.html 放在專案根目錄。

啟用 Pages

Settings ▶ Pages ▶ Source 選擇 Deploy from a branch，分支選 main。

自訂網址（可選）

在 repo 建 CNAME 寫入你的網域，或沿用 username.github.io/anemia-hub。

可選強化（無框架）

  • ✓加入 Google Analytics（或 Plausible）追蹤點擊
  • ✓加上 favicon / OG 圖，利於分享與書籤
  • ✓用 Netlify Form 或 Google 表單加上聯絡區塊
  • ✓若要內嵌外站內容，可 iframe（同源政策允許時）

色彩主題替換

只需修改上方 :root 的 CSS 變數即可。

--brand --brand-2 --panel --bg

常見問題

可以把三個內容「直接嵌入」這一頁嗎？

可以嘗試使用 <iframe> 內嵌，但需目標網頁允許。否則建議維持「摘要＋外連」做法，以保持載入速度與 SEO 乾淨。

想加上英文版/雙語版？

可複製本頁結構建立 index-en.html，或在導覽列加語言切換，連到不同頁面。

想換成更花俏的互動效果？

可以加入少量 JS 做滾動動畫（Intersection Observer）或微交互；若規模變大可改用 React / Next.js。

支援深色/淺色模式嗎？

是，已使用 color-scheme 與系統偏好自動調整。
